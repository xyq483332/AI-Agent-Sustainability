## 1. 安全目标

| 目标 | 说明 | 优先级 |
|:---|:---|:---:|
| **机密性** | 插件无法访问未授权数据 | P0 |
| **完整性** | 插件无法篡改系统状态 | P0 |
| **可用性** | 恶意插件无法耗尽系统资源 | P0 |
| **可审计性** | 所有安全相关操作可追溯 | P1 |

## 2. 威胁模型

### 2.1 威胁分类

| 威胁 | 攻击向量 | 影响 | 缓解措施 |
|:---|:---|:---|:---|
| **恶意插件** | 上传包含恶意代码的插件 | 数据泄露、系统破坏 | 沙箱隔离 + 权限验证 |
| **资源耗尽** | 插件占用过多 CPU/内存 | 系统不可用 | 资源限制 + 超时控制 |
| **权限提升** | 插件尝试获取更高权限 | 越权访问 | 最小权限原则 |
| **数据泄露** | 插件读取敏感数据 | 信息泄露 | 文件系统 ACL |
| **网络攻击** | 插件发起未授权网络请求 | 外部攻击 | 网络白名单 |
| **注入攻击** | 插件输入验证不足 | 代码注入 | 输入验证 + 参数化查询 |

### 2.2 信任边界

```
┌─── 不信任区域 ──────────────────────────────────────────┐
│  Internet / 用户输入 / 外部插件                          │
│                                                         │
│  ┌─── API 网关 (验证层) ──────────────────────────────┐  │
│  │  输入验证 | 认证 | 授权 | 速率限制                   │  │
│  │                                                     │  │
│  │  ┌─── 沙箱 (隔离层) ────────────────────────────┐  │  │
│  │  │  资源限制 | 文件 ACL | 网络 ACL | 审计日志     │  │  │
│  │  │                                               │  │  │
│  │  │  ┌─── 插件 (执行层) ──────────────────────┐  │  │  │
│  │  │  │  Plugin 代码执行                        │  │  │  │
│  │  │  └─────────────────────────────────────────┘  │  │  │
│  │  └───────────────────────────────────────────────┘  │  │
│  └─────────────────────────────────────────────────────┘  │
└───────────────────────────────────────────────────────────┘
```

## 3. 安全控制要求

### 3.1 认证与授权

| 要求 | 实现 | 状态 |
|:---|:---|:---:|
| API 认证 | JWT Token / API Key | ✅ |
| 角色管理 | RBAC (users → roles → permissions) | ✅ |
| 插件权限 | 声明式权限系统 (JSONB) | ✅ |
| 危险权限审批 | 需显式批准 `dangerous` 权限 | ✅ |

**权限列表**:

| 权限 | 说明 | 危险级别 |
|:---|:---|:---:|
| `plugin:read` | 读取插件信息 | 标准 |
| `plugin:execute` | 执行插件 | 标准 |
| `plugin:write` | 创建/修改插件 | 标准 |
| `network:access` | 网络访问 | 危险 |
| `file:read` | 文件系统读取 | 危险 |
| `file:write` | 文件系统写入 | 危险 |
| `system:admin` | 系统管理 | 危险 |

### 3.2 插件沙箱

| 要求 | 默认值 | 可配置 | 状态 |
|:---|:---:|:---:|:---:|
| CPU 限制 | 1 core | ✅ | ✅ |
| 内存限制 | 256 MB | ✅ | ✅ |
| 执行超时 | 30 秒 | ✅ | ✅ |
| 文件系统访问 | 仅白名单路径 | ✅ | ✅ |
| 网络访问 | 仅白名单 CIDR | ✅ | ✅ |
| 进程创建 | 禁止 | - | ✅ |

## 4. 安全审计

### 4.1 审计日志要求

| 事件类型 | 记录内容 | 保留期 |
|:---|:---|:---:|
| 插件加载 | plugin_id, user_id, timestamp | 90 天 |
| 插件执行 | plugin_id, duration, success | 90 天 |
| 安全验证 | plugin_id, passed, reason | 180 天 |
| 权限变更 | user_id, role_id, action | 365 天 |
| 登录事件 | user_id, ip, success | 90 天 |

### 4.2 安全事件分级

| 级别 | 说明 | 响应时间 |
|:---|:---|:---:|
| `critical` | 系统被攻破或数据泄露 | 1 小时 |
| `high` | 安全策略被绕过 | 4 小时 |
| `medium` | 可疑行为检测 | 24 小时 |
| `low` | 信息性安全事件 | 7 天 |
